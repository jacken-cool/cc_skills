# Axure 读取工作流参考

## 1. 适用目标

当用户提供 Axure 链接时，按以下顺序执行：

1. 确认读取范围（单目录 / 全量 / 批量）
2. 读取 Axure，产出需求分析文档
3. 自动判断是停下来让用户确认，还是继续测试设计
4. 若可继续，衔接飞书测试点 + CSV 用例生成

---

## 2. 范围确认话术

若用户未说明范围，优先使用以下话术确认：

- 这次要我读取单个目录 / 单个页面，还是读取全部目录 / 全部导航？
- 如果先跑小范围，我可以先读一个目录，确认格式后再扩展到全部目录。

若用户一次给出多个链接或多个导航，补充确认：

- 这次是要我把这几个链接分别独立输出，还是整理成一个批次统一汇总？
- 多个导航分支是分别出 3 份交付物，还是整合成一份总文档？

---

## 3. 自动停顿判断规则

需求分析完成后，默认自动判断是暂停还是继续。

### 3.1 应暂停等待确认的情况

- 用户明确要求先看需求分析结果。
- 原型中存在关键规则缺失或冲突，继续设计测试会明显偏航。
- 批量任务里不同链接 / 导航结构差异很大，需先确认是否拆开处理。
- 执行中发生范围扩张（单目录变多目录、多链接）。
- 待确认项直接影响测试用例预期结果。

### 3.2 可直接继续的情况

- 用户明确要求一次跑完整链路。
- 主流程、字段规则、系统边界已足够清晰。
- 待确认项不阻塞主流程测试点设计。
- 批量任务结构相似，可按统一模板稳定产出。

### 3.3 暂停时的输出要求

1. 先交付需求分析结果。
2. 明确说明为什么停下。
3. 列出影响继续产出的待确认问题。
4. 等用户确认后再继续飞书测试点和 CSV 用例。

---

## 4. 建议输出顺序

### 4.1 原型读取阶段

中间产物：
- 页面截图：`axure_page.png`
- 原始文本：`axure_text.txt`
- 需求分析：`axure_requirement_analysis.md` 或 `axure_navigation_requirement_analysis.md`

批量命名：
- `01_xxx_requirement_analysis.md`
- `02_xxx_requirement_analysis.md`
- `axure_batch_delivery_summary.md`

### 4.2 测试设计阶段

- 飞书测试点（Markdown）：`xxx_test_points_feishu.md`
- 飞书测试点（.mm）：`xxx_test_points_feishu.mm`
- CSV 测试用例：`xxx_test_cases.csv`

批量命名：
- `01_xxx_test_points_feishu.md`
- `01_xxx_test_points_feishu.mm`
- `01_xxx_test_cases.csv`

---

## 5. 需求分析重点

至少抽取以下内容：

- 页面目标
- 导航位置
- 字段与规则
- 主流程
- 跨系统关系
- 异常 / 风险 / 待确认项
- 批量任务补充每个链接 / 导航的差异说明

---

## 6. 典型提示词

### 一次跑完整链路

```
请读取这个 Axure 链接，先让我选择读取单个目录还是全部目录，然后一次输出需求分析文档、飞书思维笔记格式测试点文档（Markdown 或 .mm 格式）和 CSV 测试用例。
```

### 自动判断是否停在需求分析阶段

```
请处理这个 Axure 链接。若需求分析已足够清晰，就继续输出飞书测试点（Markdown 或 .mm 格式）和 CSV 用例；如果关键规则还不清楚，就先停在需求分析阶段让我确认。
```

### 指定全量读取

```
请读取这个 Axure 导航全部目录，直接产出需求分析、飞书测试点和 CSV 用例。
```

### 指定单目录读取

```
只读取"ITP配置新增字段"这个目录，然后生成需求分析、飞书测试点和 CSV 用例。
```

### 多链接批量处理

```
批量处理这 3 个 Axure 链接，分别输出需求分析、飞书测试点和 CSV 用例；如果某个链接规则不清晰，就单独停在需求分析阶段等我确认。
```

---

## 7. 注意事项

1. 不要跳过需求分析，直接从 Axure 拼测试用例。
2. 对无法从原型确认的规则统一标注"待确认"。
3. 若用户已有历史需求文档或测试点，优先增量整合，不要无故覆盖。
4. 若用户只想要某一阶段结果，可在全链路中止步于对应阶段。
5. 批量任务中优先保证每个链接 / 导航分组的对应关系清晰，避免交叉污染。

---

## 8. Axure 源文件直接读取方法（curl 方案）

### 8.1 适用场景

- 内网 Axure 链接（agent-browser 或 WebFetch 无法访问）
- Chrome 沙箱/环境问题导致浏览器自动化失败
- 无需渲染动态交互效果，仅需读取静态文案

### 8.2 读取步骤

#### 第一步：定位 Axure 发布目录

Axure 原型发布后，所有文件部署在同一个 HTTP 目录下。用户提供的 URL 中的 `#` 前面部分即为根目录：

```
URL: http://xxx.com/axure/项目名称/页面名称/#g=1&id=xxx

根目录: http://xxx.com/axure/项目名称/页面名称/
```

#### 第二步：获取页面清单（data/document.js）

从根目录抓取 `data/document.js`：

```bash
curl -s "http://xxx.com/axure/项目名称/页面名称/data/document.js"
```

该文件包含 `$axure.loadDocument(...)` 调用，其中可解析出所有页面的清单（`sitemap → rootNodes` 部分）。从中提取：

- **页面 ID**（`id` 字段）
- **页面名称**（`pageName` 字段，包含中文标题）
- **页面类型**（`type` 字段，通常为 `Wireframe`）
- **页面 URL**（`url` 字段，格式为 `_{页面名称}.html`）

示例如下：

```javascript
r="rootNodes",s="id",t="ichdbc",u="pageName",v="#169779人设日签支持排序",w="type",x="Wireframe",y="url",z="_169779人设日签支持排序.html",A="children"
```

此例表示 2 个页面：
1. ID=`ichdbc`, 名称=`#169779人设日签支持排序`, URL=`_169779人设日签支持排序.html`
2. ID=`4dtx2p`, 名称=`#169779小程序：日签 (支持排序）`, URL=`_169779小程序：日签（支持排序）.html`

#### 第三步：逐页抓取 HTML 内容

根据第二步发现的每个页面 URL，依次抓取：

```bash
curl -s "http://xxx.com/axure/项目名称/页面名称/_{页面名称}.html"
```

#### 第四步：从 HTML 提取文案

Axure 原型页面 HTML 中，真正的内容包裹在 `div_text` 的 `p > span` 结构中。提取方式：

1. 定位 `<div id="base" class="">` 容器
2. 查找该容器下所有带有 `.text` 类的 `div`（即 `div_text`）
3. 从其中的 `<p><span>` 元素提取可见文本
4. 对于 `<ul>` 列表结构，逐项提取文字

文本提取示例：

```html
<!-- 需求说明面板 -->
<div id="u10_text" class="text ">
  <p><span style="...">1、日签模板可关联3个分类</span></p>
  <p><span style="...">2、轮播展示顺序...</span><span style="...">（蓝色=新增，删除线=废弃）</span></p>
</div>

<!-- 流程图节点 -->
<div id="u11_text" class="text ">
  <p><span>微信用户访问新平衡+小程序</span></p>
</div>

<!-- 表格单元格 -->
<div id="u32_text" class="text ">
  <p><span>养生理念</span></p>
</div>
```

#### 第五步：还原业务逻辑

从提取的文本中梳理：

- **需求规则**：从需求说明面板中提取编号列表
- **流程图分支**：从菱形/矩形节点中提取判断条件和分支路径
- **表格数据**：从表格单元格中还原行列表结构
- **标记识别**：注意 `color:#0000FF`（蓝色=新增规则）、`text-decoration:line-through`（删除线=已废弃）等样式标记

### 8.3 注意事项

1. `data/document.js` 内容经过高度混淆压缩，只需从中提取 `pageName`、`url`、`id` 等页面元信息即可，不需要解析完整结构
2. 部分页面 URL 中的中文需要进行 URL 编码或直接使用原始中文（取决于服务器配置）
3. 如果页面 HTML 中的 `div_text` 为空（`display:none; visibility: hidden`），则该文本可能在原型中不可见，仅为 Axure 内部占位
4. 对于流程图，连接的箭头和路径标注通常在线性 SVG 中，而关键逻辑在节点（菱形/矩形）的文本中
5. 表格数据注意还原表头行，避免列数据错位
