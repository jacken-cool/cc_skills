# 接口测试用例设计方法论

## 场景分类（6类覆盖矩阵）

| 序号 | 类别 | 说明 | 优先级 |
|:---:|------|------|:------:|
| 1 | 入口/正常场景 | 所有入口的参数组合，各分类正确返回 | P0 |
| 2 | 降级逻辑 | 分类下线、无数据、编码不存在时的回退行为 | P0 |
| 3 | 业务规则 | 排序、数量阈值、循环算法、条件判断 | P0 |
| 4 | 响应结构验证 | 字段存在性、类型、新增字段 | P0 |
| 5 | 参数校验 | 必填、类型错误、边界值、空字符串 | P1-P2 |
| 6 | 异常/安全 | XSS、不存在的资源、超长输入 | P1-P2 |

## 各场景设计要点

### 1. 入口/正常场景
- 每个入口/分类编码一个用例
- 验证返回数据属于对应分类
- 包含默认值场景（不传可选参数时的行为）
- 示例：4 个 tab 入口各一条 + 默认入口一条 = 5 条

### 2. 降级逻辑
- A 分类下线 → 降级到 B 分类
- A 分类上线但无数据 → 降级到 B 分类
- 所有分类不可用 → 返回空
- 每种降级路径一条用例

### 3. 业务规则
- 引导问题置顶（sort=0）
- 数量阈值判断（>3 条显示换一批，≤3 条不显示）
- 循环展示算法（4条、5条、6条边界）
- 每批数量限制（最多3条）
- 每条规则正反两条覆盖

### 4. 响应结构验证
- 新增字段存在性（suggestId、name、suggestCateCode 等）
- 字段类型验证（boolean 非 string、number 非 string）
- 每条用例用 `!= null` 验证字段存在

### 5. 参数校验
- 必填参数缺失 → 返回错误码
- 类型错误（number 传 string）→ 根据实现校验或容错
- 边界值：0、负数、最大值、空字符串
- 编码不存在 → 降级或返回错误

### 6. 异常/安全
- XSS 注入字符串过滤
- 不存在的 agentCode
- 服务不崩溃、不返回敏感信息

## 断言生成规则

### 成功响应断言
```
$.response.code = "00000"                     # 成功码
$.response.data.xxx != null                   # 字段存在
$.response.data.xxx[0].field = "期望值"        # 字段值匹配
$.response.data.xxx.length() = 3              # 数组长度
$.response.data.xxx.length() > 2              # 数组长度下限
$.response.data.xxx.length() <= 3             # 数组长度上限
$.response.data.xxx[0].showMoreSuggestes = true  # boolean 值
```

### 失败响应断言
```
$.response.code != "00000"      # 非成功码
$.response.success = false      # 失败标识
$.response.msg != null          # 有错误提示
```

### 纯文本断言（非 JSONPath）
当断言内容是行为描述而非具体字段校验时，path 和 op 留空，值放在断言值列：
- `第2批内容为[第4条,第1条,第2条]（4,1,2）`
- `接口不返回script标签内容`
- `轮询不重复遗漏`

## 优先级分配

| 优先级 | 场景 |
|:------:|------|
| P0 | 主流程闭环、降级逻辑、核心业务规则、响应结构验证 |
| P1 | 边界值、必填校验、不存在资源、字段遗漏 |
| P2 | XSS、类型边界、超大数值、并发 |

## 用例设计自检清单

生成用例后检查：
- [ ] 所有独立入口是否覆盖
- [ ] 所有降级路径是否覆盖（下线/无数据/兜底）
- [ ] 所有业务规则是否正反覆盖
- [ ] 响应结构验证是否包含所有新增字段
- [ ] 必填参数是否校验
- [ ] 是否至少有一条异常安全用例
- [ ] 断言是否全部使用 `$.response.` 前缀
- [ ] 断言值是否无外层双引号
